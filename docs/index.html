Webrig

A general-purpose browser with audio routing. Load any URL, stream audio to Discord.

Features

Everything in one window

Webrig is a controllable Electron browser. The Discord audio bridge is just one output — the core is a full Chromium browser with automation built in.

Loads any URL

YouTube, SoundCloud, Twitch, voice calls — any web page loads and plays as normal. Change URLs on the fly via the built-in navbar.

Modern Chrome compatibility

Built on Chromium and configured to be compatible with sites that expect a modern Chrome browser — User-Agent, navigator properties, and standard web APIs all in order.

CDP automation

Chrome DevTools Protocol exposed on port 9229. Use agent-browser to navigate, screenshot, fill forms, and automate anything.

Discord audio streaming

All audio playing in the window — video, music, calls — is captured via Web Audio API and streamed live to a Discord voice channel via an official bot.

Live navigation bar

Injected navbar lets you type a new URL and reload the window without restarting the app. Change content on the fly without touching config files.

Bidirectional audio

Outbound: window audio → Opus → Discord. Inbound: Discord voice → decoded PCM → Web Audio playback. Full two-way voice bridge.

How it works

Audio pipeline in four steps

Webrig runs as a single Electron process. No separate bot process, no sockets — the discord.js bot shares the same Node.js event loop as the browser window.

1

Load URL

Electron opens a BrowserWindow at TARGET_URL. The Chromium window is configured for broad site compatibility.

2

Tap audio

Preload script patches AudioContext and intercepts all audio elements. Every sound is captured via Web Audio API.

3

Encode to Opus

Raw PCM Float32 frames are converted to s16le and encoded to Opus at 48 kHz via prism-media.

4

Stream to Discord

Opus frames flow into a discord.js AudioPlayer resource and are broadcast live into the configured voice channel.

Installation

Get started in minutes

Download the prebuilt Windows installer or clone and run from source. Node.js 18+ required for source builds.

Option A — Prebuilt installer (Windows)

Download latest release

Option B — Build from source

terminal 
git clone https://github.com/AnEntrypoint/webrig
cd webrig
npm install --legacy-peer-deps
cp .env.example .env
# edit .env with your values
npm start

CDP automation (agent-browser)

terminal 
# Install agent-browser once
npm i -g @vercel-labs/agent-browser

# While Webrig is running:
agent-browser snapshot
agent-browser open https://youtube.com
agent-browser screenshot

PM2 (optional, keeps running after terminal close)

terminal 
pm2 start ecosystem.config.cjs
pm2 logs webrig
Configuration

.env variables

Copy .env.example to .env and fill in your values. Guild ID and Channel ID: enable Developer Mode in Discord settings, right-click server/channel, copy ID.

Variable Description Default
DISCORD_BOT_TOKEN Discord bot token. Bot must have CONNECT + SPEAK permissions in the target voice channel. —
TARGET_URL URL to load on startup. Any web page — YouTube, SoundCloud, a web app, etc. https://example.com
GUILD_ID Discord server (guild) ID. Right-click the server icon → Copy Server ID. —
CHANNEL_ID Voice channel ID to join. Right-click the channel → Copy Channel ID. —
CDP_PORT Chrome DevTools Protocol port. Connect with agent-browser connect <port>. 9229
